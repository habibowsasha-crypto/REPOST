# FIX REPORT - ROUTE_MAP для отдельных каналов

Добавлена отдельная функция маршрутизации обычных каналов без тем.

## Новая переменная

```env
ROUTE_MAP=-100SOURCE1>-100TARGET,-100SOURCE2>-100TARGET
```

## Что делает

- Старые переменные `SOURCE_CHATS`, `TARGET_CHAT`, `SOURCE_TOPIC_MAP`, `TOPIC_ROUTE_MAP` не ломаются.
- `ROUTE_MAP` добавляет отдельные маршруты `источник -> получатель`.
- Можно отправлять 2 новых канала в отдельный канал, не трогая старый VIP-хаб.
- Поддерживается copy/forward согласно `COPY_MODE`.
- Синхронизация edit/delete продолжает работать через сохраненную SQLite-связку.

## Пример

```env
ROUTE_MAP=-1001111111111>-1003333333333,-1002222222222>-1003333333333
```

Расшифровка:

```text
Канал 1 -> отдельный канал
Канал 2 -> отдельный канал
```

## Важно

Тестировать нужно на новых постах после redeploy. Старые посты не имеют новых связок в базе.
