TgBlaster DM + AI Dialog v1.0.5
Audit and runtime hardening - no new restrictions

Исправлено:

1. Убрана ошибочная зависимость рабочего списка от прав администратора/владельца.
   Обычный участник может использовать доступную группу или супергруппу.
2. Нормализованы Bot API ID вида -100... и старые значения после abs().
3. Для закрытых супергрупп используется сохранённый access_hash после рестарта.
4. Исправлена связь общего каталога, найденных групп и рабочих групп аккаунта.
5. Ручное исключение группы сохраняется после повторной синхронизации.
6. Активные DM-задачи согласуются с удалением недоступных рабочих групп.
7. DM отправляет сообщение через сохранённый объект User, а не только числовой ID.
8. Исправлены дубли мониторов и повторная обработка одного входящего AI-сообщения.
9. Dry-run AI больше не увеличивает счётчик реальных ответов и не закрывает диалог.
10. Исправлена миграция старых дубликатов AI-диалогов без потери истории сообщений.
11. Обычная рассылка использует единый hardened runtime и не создаёт две задачи на одну группу.
12. Исправлен диапазон случайного интервала: APScheduler получает base=min и jitter=max-min.
13. Исправлено возобновление рассылки только с изображением.
14. Исправлен ложный успех при отсутствии рабочих групп.
15. Ошибка одного аккаунта больше не отменяет запуск рассылки для остальных аккаунтов.
16. Добавлена дедупликация записей broadcasts и уникальный индекс user_id/group_id.
17. При удалении аккаунта останавливаются scheduler-задачи, DM-мониторы и очищается копия session_string в dm_tasks.
18. Удалён неиспользуемый legacy-код callback cache и лишние state-переменные.
19. Некорректный FIRST_DM_VIP_WEIGHT_PERCENT больше не ломает DM-worker.
20. Сохранены команда «Меню» и редактирование inline-меню в одном сообщении.

Новых ограничений поведения не добавлено.
