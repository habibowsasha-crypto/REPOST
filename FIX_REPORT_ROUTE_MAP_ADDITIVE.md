# FIX REPORT - ROUTE_MAP additive routing

## Что исправлено

Раньше, если один и тот же источник был указан и в `SOURCE_CHATS`, и в `ROUTE_MAP`, бот отправлял пост только по `ROUTE_MAP`.

Теперь `ROUTE_MAP` работает как дополнительный маршрут, а не как замена старого маршрута.

## Новая логика

Если источник есть в `SOURCE_CHATS` и в `ROUTE_MAP`, один пост будет отправлен сразу в оба направления:

```text
SOURCE_CHATS -> TARGET_CHAT / нужная тема
ROUTE_MAP    -> отдельный канал / отдельная цель
```

## Также исправлено

- ключ дублей теперь учитывает точку назначения;
- один и тот же пост может быть скопирован в несколько целей;
- edit-синхронизация обновляет все копии поста;
- delete-уведомления отправляются по всем сохранённым копиям;
- старые переменные `SOURCE_CHATS` и `TARGET_CHAT` не ломаются.

## Пример

```env
SOURCE_CHATS="-1001111111111:12"
TARGET_CHAT="-1003732726013"
ROUTE_MAP="-1001111111111>-1002222222222"
```

Результат:

```text
-1001111111111 -> -1003732726013 тема 12
-1001111111111 -> -1002222222222 без темы
```

## Важно

Для старых постов, которые уже были обработаны до фикса, повторная отправка может блокироваться старой базой дублей. Для чистого теста используй новый пост после redeploy.
