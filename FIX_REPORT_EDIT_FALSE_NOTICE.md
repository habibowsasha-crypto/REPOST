# Fix report — edit false notice

Исправлена проблема, когда бот отправлял уведомление:

> ⚠️ Источник изменил пост. Текст/подпись обновлены, если это было возможно.

даже если видимых изменений в посте не было.

## Что изменено

- Добавлено хранение исходного текста/подписи в SQLite `message_map`.
- При событии `MessageEdited` бот теперь сравнивает старый и новый текст.
- Если текст/подпись не изменились — событие игнорируется.
- Если текст/подпись реально изменились — бот редактирует свою копию.
- Уведомление теперь отправляется только если бот не смог автоматически обновить копию.
- После успешного редактирования уведомление больше не отправляется.

## Важно

Для уже скопированных старых постов, у которых в базе ещё нет `source_text`, первое edit-событие может быть обработано как реальное изменение. Для новых постов после установки этой версии сравнение будет работать корректно.
