# UPDATE REPORT - Delete/Edit Sync

## Что добавлено

1. Сохранение связки оригинального поста и копии.
   - Таблица `message_map` в SQLite.
   - Формат связи: `source_chat_id + source_message_id -> target_chat_id + target_message_id`.
   - Связки сохраняются после рестарта Railway.

2. Реакция на удаление поста в источнике.
   - Новый handler: `events.MessageDeleted`.
   - Если исходный пост удалён, бот находит свою копию и отвечает на неё сообщением:
     `⚠️ Источник удалил этот пост`.
   - Повторные уведомления по одному удалению блокируются через таблицу `deletion_notices`.

3. Реакция на редактирование поста в источнике.
   - Новый handler: `events.MessageEdited`.
   - Если исходный текст/подпись изменены, бот редактирует свою копию.
   - Если пост с медиа и Telegram не даёт надёжно заменить медиа, бот обновляет текст/подпись и отправляет предупреждение ответом.

4. Новые ENV-переменные:
   - `SYNC_DELETES=true`
   - `SYNC_EDITS=true`
   - `DELETE_NOTICE_TEXT=⚠️ Источник удалил этот пост`
   - `EDIT_MEDIA_NOTICE_TEXT=⚠️ Источник изменил пост. Текст/подпись обновлены, если это было возможно.`

## Важные ограничения

1. Синхронизация удалений/изменений работает только для постов, которые были скопированы после установки этой версии.
   - Старые посты не имеют записанной связки в `message_map`.

2. Для текста и подписи синхронизация работает нормально.

3. Для замены медиа Telegram/Telethon не всегда позволяет надёжно отредактировать уже отправленное вложение.
   - Безопасная логика: обновить текст/подпись и отправить уведомление.

4. Для альбомов используется best-effort mapping.
   - Если Telegram вернул список отправленных сообщений один к одному, каждое фото/видео получает свою связку.
   - Если нет, бот сохраняет связь на первое сообщение альбома.

5. Лучше использовать `COPY_MODE=copy`.
   - В `forward` режиме редактирование пересланных сообщений может иметь ограничения Telegram.

## Обновление: маршрутизация источников по темам

Добавлена поддержка отправки разных источников в разные темы одной forum-группы.

Вариант 1 - прямо в `SOURCE_CHATS`:

```env
TARGET_CHAT=-1003732726013
SOURCE_CHATS=-1003772161356:10,-1003849424671:12,-1003904203455:4
```

Вариант 2 - отдельно через `SOURCE_TOPIC_MAP`:

```env
TARGET_CHAT=-1003732726013
SOURCE_CHATS=-1003772161356,-1003849424671,-1003904203455
SOURCE_TOPIC_MAP=-1003772161356:10,-1003849424671:12,-1003904203455:4
```

`TARGET_THREAD_ID` оставляй пустым, если используешь разные темы для разных источников.
