## Важно знать про хранение сессий

Сессионные строки Telethon хранятся в `sessions.db` **в открытом виде** (plaintext). Это значит:

- Тот, у кого есть доступ к этому файлу, может войти в Telegram-аккаунты, добавленные в бот, без пароля.
- `sessions.db` уже в `.gitignore` — убедись, что он туда не попал.
- Никому не передавай этот файл и не кладя в облако без шифрования.
- На продакшне ограничь права на чтение файла (`chmod 600 sessions.db`).

Это известная слабость архитектуры. В планах — зашифрованное хранилище сессий (см. ниже).

## Известные проблемы (planned hardening)

| Проблема | Статус |
|---|---|
| Сессии хранятся в plaintext в SQLite | Открыто — нужно шифрование |
| Нет ротации секретов / audit log доступа | Не реализовано |
| Доступ к боту по whitelist (ADMIN_ID_LIST), но без 2FA на уровне бота | Приемлемо для личного использования |

Если хочешь помочь с шифрованием сессий — добро пожаловать в PR.

---

## Общие рекомендации

- Держи `.env` с секретами только локально или в защищённом secret manager.
- Не запускай бота от root.
- Если деплоишь на сервер — изолируй в Docker с минимальными правами.
