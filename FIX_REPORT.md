# FIX REPORT - Telegram Userbot Reposter

## Что исправлено

1. Добавлена централизованная очередь обработки сообщений.
   - Старый риск: несколько handler'ов могли отправлять посты параллельно и ломать порядок.
   - Новая логика: все посты идут через `asyncio.Queue`, обрабатываются одним worker'ом строго по очереди.

2. Исправлена задержка между отправками.
   - Старый риск: `DELAY_SECONDS` работал внутри параллельных handler'ов и не давал настоящего throttling.
   - Новая логика: отправки идут последовательно, задержка стала глобальной по факту.

3. Добавлена обработка web preview.
   - Старый риск: пост со ссылкой мог восприниматься как media и ломаться на отправке/скачивании.
   - Новая логика: `MessageMediaWebPage` копируется как текст с link preview.

4. Добавлена persistent-защита от дублей.
   - Старый риск: после рестарта in-memory кэш очищался.
   - Новая логика: обработанные сообщения сохраняются в SQLite.

5. Добавлена preflight-проверка при старте.
   - Проверяется доступность `SOURCE_CHATS`.
   - Проверяется доступность `TARGET_CHAT`.
   - Проверяется доступность `LOG_CHAT`, если указан.

6. Добавлена защита от бесконечного репоста.
   - Если `TARGET_CHAT` случайно указан в `SOURCE_CHATS`, бот падает на старте с понятной ошибкой.

7. Улучшена диагностика `SESSION_STRING`.
   - Поврежденная строка сессии теперь дает понятную ошибку.
   - В `main.py` убран интерактивный `client.start()`, чтобы Railway не зависал на вводе телефона/кода при плохой сессии.

8. Добавлена поддержка Telegram topics.
   - Новый ENV: `TARGET_THREAD_ID`.
   - Лучше использовать с `COPY_MODE=copy`.

9. Улучшены ENV-настройки.
   - Добавлены `QUEUE_MAXSIZE`, `PERSIST_PROCESSED`, `PROCESSED_DB_PATH`, `LINK_PREVIEW`, `PRESERVE_BUTTONS`, `MAX_FLOOD_WAIT_SECONDS`, `LOG_LEVEL`.

10. Очищен готовый архив.
   - `__pycache__` и `.pyc` удалены.

## Что по-прежнему является ограничением Telegram, а не багом проекта

- Protected content может не копироваться.
- Опросы, dice, контакты, геолокации и часть нестандартных вложений не гарантируются как полная копия.
- Callback-кнопки чужих ботов не всегда можно пересоздать.
- Изменения/удаления синхронизируются только для постов, которые были скопированы после установки версии с `message_map`. Старые посты без сохраненной связки не синхронизируются.
- `TARGET_THREAD_ID` наиболее надежен в `COPY_MODE=copy`.
