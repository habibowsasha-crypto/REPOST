# TgBlaster DM + AI Dialog

Версия: **1.0.6**

Telegram-проект с админским ботом, подключаемыми пользовательскими аккаунтами Telethon, обычной рассылкой, DM-задачами, случайными первыми сообщениями и AI-продолжением личных диалогов.

## Главное в этой версии

- Меню открывается командами `/start`, `/menu` и сообщением `Меню`.
- Inline-кнопки обновляют текущее сообщение меню, а не создают новое.
- Бот синхронизирует группы и супергруппы, доступные подключённому аккаунту.
- **Админство или владение группой не требуется. Обычного участия достаточно**, если аккаунт видит сообщения группы.
- Поддерживаются закрытые супергруппы: сохраняется `access_hash`, поэтому они восстанавливаются после перезапуска Railway.
- DM-задача выбирает рабочие группы аккаунта и слушает новые сообщения в них.
- Случайное первое DM выбирается из категорий `soft`, `trading`, `vip` или `all`.
- AI-слой работает в компактном сценарии: первое случайное DM → один полный информационный ответ со ссылкой → завершение диалога.
- После отправки ссылки бот не продолжает переписку. При явном отказе допускается только одно короткое извинение.

## Установка на Railway

1. Подключите Volume к пути `/data`.
2. Добавьте Variables из `.env.example`, заменив тестовые значения реальными.
3. Start command уже указан в `railway.json`: `python main.py`.
4. После запуска отправьте боту `Меню`.
5. Добавьте Telegram-аккаунт, затем откройте его карточку и нажмите `Найти группы аккаунта`.

### Случайное первое сообщение

```env
DM_FIRST_MESSAGE_RANDOM=true
FIRST_DM_MODE=random
FIRST_DM_CATEGORY=all
FIRST_DM_VIP_WEIGHT_PERCENT=30
FIRST_DM_TEMPLATES=
FIRST_DM_TEMPLATES_FILE=
```

При пустых пользовательских шаблонах используются встроенные варианты. Старый текст `/dm_post` остаётся fallback-значением.

### AI-диалоги

```env
AI_DM_ENABLED=true
AI_DM_DRY_RUN=true
OPENAI_API_KEY=
AI_MODEL=gpt-4o-mini
AI_OFFER_INTRO_MAX_TOKENS=48
AI_REPLY_ONLY_KNOWN_DIALOGS=true
AI_DAILY_DIALOG_LIMIT=0
AI_REPLY_DELAY_MIN_SECONDS=15
AI_REPLY_DELAY_MAX_SECONDS=60
AI_POST_OFFER_APOLOGY=Понял, извини за беспокойство. Больше писать не буду 🙌
```

`AI_DM_DRY_RUN=true` сохраняет тестовый черновик в базе и логах, но не отправляет его в Telegram. Для реальных ответов установите `false`.

Компактный сценарий фиксирован:

```text
1. DM-модуль отправляет случайное первое сообщение.
2. После ответа пользователя бот отправляет один полный блок о PIRATE VIP FREE:
   - все 5 источников;
   - задержка дублирования до 0.5 секунды;
   - совокупная стоимость доступов более 1000$;
   - бесплатный доступ;
   - точная ссылка https://telegram.me/+pvPjmt2KW_QyZTAy.
3. Диалог закрывается. Новые сообщения после ссылки игнорируются.
4. При явном отказе допускается одно извинение, после чего ответов больше нет.
```

## Работа с группами

Путь в меню:

```text
Меню → Мои аккаунты → нужный аккаунт → Найти группы аккаунта
```

Синхронизация:

- получает диалоги именно подключённого пользовательского аккаунта;
- оставляет группы и супергруппы;
- исключает обычные каналы-витрины;
- сохраняет публичные и закрытые группы;
- показывает роль аккаунта только как справочную информацию;
- автоматически добавляет впервые найденные доступные группы в рабочий список;
- сохраняет ручное исключение группы при следующих синхронизациях.

## Хранение данных

При Volume `/data` сохраняются:

- `/data/sessions.db` - SQLite;
- `/data/bot.session` - сессия админского бота;
- `/data/media` - загруженные изображения;
- `/data/logs` - ротационные логи.

Не помещайте `.env`, реальные токены, API-ключи, `.session`, базы и логи в Git или ZIP.

## Изменение v1.0.7

При `DM_FIRST_MESSAGE_RANDOM=true` или `FIRST_DM_MODE=random` мастер DM больше не просит ручной текст. После выбора чатов он сразу переходит к интервалу повтора. При выключенном случайном режиме старый ручной ввод текста сохраняется.

## Исправление v1.0.8

Исправлен сценарий повторного AI-цикла: если этому же пользователю ранее уже было отправлено VIP-предложение, новый реально отправленный первый DM снова открывает AI-диалог. Ответ пользователя больше не игнорируется из-за старого статуса `completed` или старой ссылки в истории.

После отправки предложения в текущем цикле поведение не изменилось: диалог завершается, обычные последующие сообщения игнорируются, а при явном отказе допускается только согласованное однократное извинение.

## v1.0.10 — очистка неактуальных DM-задач

В разделе `📋 DM-задачи` появилась кнопка `🧹 Очистить неактуальные`. Перед удалением бот показывает подтверждение. Удаляются только остановленные задачи (`is_active = 0`) и их технические строки очереди/логов; активные задачи, аккаунты и история AI-диалогов не затрагиваются.



## v1.0.11 — гарантированная видимость очистки DM-задач

- Кнопка `🧹 Очистить неактуальные (N)` теперь всегда отображается на экране `📋 DM-задачи`.
- Количество остановленных задач считается отдельным SQL-запросом.
- Добавлена кнопка `🔄 Обновить список`.
- Добавлен прямой пункт очистки в главном меню.
- Добавлена резервная команда `/dm_cleanup`.
- Удаление по-прежнему затрагивает только остановленные задачи после подтверждения.
