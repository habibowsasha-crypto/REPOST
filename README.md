# TgBlaster DM + AI Dialog

Версия: **1.0.18**

Telegram-проект с админским ботом, подключаемыми пользовательскими аккаунтами Telethon, обычной рассылкой, DM-задачами, случайными первыми сообщениями и AI-продолжением личных диалогов.

## Главное в этой версии

- Меню открывается командами `/start`, `/menu` и сообщением `Меню`.
- Inline-кнопки обновляют текущее сообщение меню, а не создают новое.
- Бот синхронизирует группы и супергруппы, доступные подключённому аккаунту.
- **Админство или владение группой не требуется. Обычного участия достаточно**, если аккаунт видит сообщения группы.
- Поддерживаются закрытые супергруппы: сохраняется `access_hash`, поэтому они восстанавливаются после перезапуска Railway.
- DM-задача выбирает рабочие группы аккаунта и слушает новые сообщения в них.
- Случайное первое DM выбирается из категорий `soft`, `trading`, `vip` или `all`.
- AI-слой после ответа пользователя работает как персонаж **Максим**: учитывает текущий диалог, отвечает на вопросы, обсуждает VIP-доступы и естественно подводит к бесплатной ссылке.
- Максим честно говорит, что занимается привлечением трафика за зарплату; при вопросе о выгоде раскрывает модель с платным расширенным доступом почти к 50 СНГ- и западным источникам.
- Прямое разрешение «скинь ссылку» не обязательно: после уместного прогрева Максим может объяснить ценность и сразу добавить ссылку.
- При подозрении в обмане Максим не спорит и не оправдывается, а снимает давление. При прямом отказе извиняется один раз, сохраняет Telegram ID в постоянный opt-out и больше не запускает для него первый DM.
- После отправки ссылки Максим может один раз ответить на дополнительный вопрос пользователя, затем диалог логически завершается. Если ответа нет, завершение выполняется по настраиваемому тайм-ауту.

## Установка на Railway

1. Подключите Volume к пути `/data`.
2. Добавьте Variables из `.env.example`, заменив тестовые значения реальными.
3. Start command уже указан в `railway.json`: `python main.py`.
4. После запуска отправьте боту `Меню`.
5. Добавьте Telegram-аккаунт, затем откройте его карточку и нажмите `Найти группы аккаунта`.

### Контакты и завершённые диалоги

```env
DM_DIALOG_ABANDON_HOURS=72
DM_DIALOG_POST_LINK_COMPLETE_HOURS=72
```

До ссылки отсутствие ответа в течение 72 часов переводит цикл в `abandoned`: бот перестаёт следить за ним, но не создаёт защиту от будущего контакта. После ссылки тайм-аут переводит цикл в `completed` и защищает пользователя от повторного первого DM только для того подключённого аккаунта, который завершил разговор.

В меню `📊 Контакты и диалоги` доступны общая статистика, статистика по исходным чатам и подтверждаемая кнопка `🧹 Разрешить повторный контакт`. Очистка не отправляет сообщения сразу, не восстанавливает старую очередь и не затрагивает постоянный opt-out. Новый первый DM становится возможен только после нового сообщения пользователя в одном из отслеживаемых этим аккаунтом чатов.

`AI_DM_DRY_RUN=true` сохраняет тестовые черновики в базе и логах, но не отправляет их в Telegram. Для реальных ответов установите `false`.

Контекстная воронка:

```text
1. Существующий DM-модуль отправляет первое случайное сообщение без изменений.
2. После ответа пользователя Максим читает историю текущего цикла, отвечает по смыслу и переходит к теме VIP.
3. Максим выясняет опыт/сомнения, обсуждает проблему оплаты неизвестной випки и честно объясняет свою роль трафера.
4. При обычном интересе или нейтральном продолжении Максим объясняет простыми словами: есть бесплатная Telegram-группа, программа моментально копирует туда посты из 6 платных закрытых VIP-каналов известных трейдеров, а отдельные доступы стоят сотни долларов. Затем даёт точную ссылку:
   https://telegram.me/+pvPjmt2KW_QyZTAy
5. Прямое согласие «скинь ссылку» не обязательно.
6. На вопрос «в чём выгода?» Максим сообщает: он получает зарплату за трафик, а создатель позже продаёт расширенный доступ почти к 50 СНГ- и западным VIP-источникам.
7. На «обман/наёб/скам» Максим не спорит: говорит, что человека никто не заставляет, можно посмотреть или отказаться. При просьбе не писать Telegram ID навсегда сохраняется в постоянном opt-out.
8. После ссылки Максим отвечает на одно дополнительное сообщение по смыслу и завершает диалог. При отсутствии ответа диалог завершается по тайм-ауту. Прямой отказ в любой момент сохраняет постоянный opt-out.
```

Обычная ветка занимает около 4-6 сообщений Максима. `AI_MAX_FOLLOWUP_MESSAGES=7` ограничивает обычные follow-up-сообщения. Если необычный старый цикл дошёл до предела без ссылки, допускается ровно одна финальная реплика со ссылкой или естественным закрытием, чтобы не бросать пользователя посреди разговора. Это никак не меняет текст первого DM, формат очереди, интервалы или случайный выбор первого сообщения.

## Работа с группами

Путь в меню:

```text
Меню → Мои аккаунты → нужный аккаунт → Найти группы аккаунта
```

Синхронизация:

- получает диалоги именно подключённого пользовательского аккаунта;
- оставляет группы и супергруппы;
- исключает обычные каналы-витрины;
- сохраняет публичные и закрытые группы;
- показывает роль аккаунта только как справочную информацию;
- автоматически добавляет впервые найденные доступные группы в рабочий список;
- сохраняет ручное исключение группы при следующих синхронизациях.

## Хранение данных

При Volume `/data` сохраняются:

- `/data/sessions.db` - SQLite;
- `/data/bot.session` - сессия админского бота;
- `/data/media` - загруженные изображения;
- `/data/logs` - ротационные логи.

Не помещайте `.env`, реальные токены, API-ключи, `.session`, базы и логи в Git или ZIP.

## v1.0.12 — название исходного чата во втором сообщении

После первого ответа пользователя второе сообщение теперь строится вокруг названия именно того отслеживаемого чата, где пользователь написал перед первым DM. Используются локальные вариативные шаблоны, например:

```text
Слушай, а у тебя есть VIP от «Название чата»?
Ты давно сидишь в «Название чата»?
Как думаешь, у «Название чата» реально высокий винрейт?
```

Название сохраняется в текущем AI-цикле при успешной отправке первого DM. Если Telegram не отдал название чата, используется прежнее короткое продолжение разговора без пустой подстановки.

Сценарий остаётся трёхсообщенческим:

```text
1. Случайное первое DM.
2. После ответа пользователя — короткий вопрос с названием исходного чата.
3. После следующего ответа — вариативное предложение PIRATE VIP FREE с обязательным блоком и точной ссылкой.
4. После ссылки диалог завершается; допускается только одно извинение при явном отказе.
```

Триггер первого DM, очередь, интервалы, очистка DM-задач и правила остановки AI не изменялись.

## v1.0.14 — контекстная AI-воронка «Максим»

- Добавлена постоянная личность: Максим, живой трафик-менеджер/продажник, а не трейдер-гуру.
- Модель получает историю текущего первого-DM цикла, исходный чат, этап и конкретную задачу ответа.
- Вместо случайного вопроса вне контекста работает управляемая воронка: VIP-вопрос → проблема оплаты → честный оффер → ссылка.
- На вопросы пользователя сначала даётся ответ по смыслу.
- Вопросы о выгоде раскрывают бизнес-модель: трафик за зарплату и расширенный вариант почти с 50 СНГ- и западными источниками.
- Подозрения «обман/наёб/скам» получают спокойный ответ без давления и без автоматической остановки.
- Прямой отказ или направленная агрессия приводят к одному извинению и остановке AI-follow-up.
- Ссылка может быть дана без отдельной просьбы пользователя, когда она уже уместна в контексте.
- Добавлены локальные fallback-ответы: воронка продолжает работать даже без OpenAI или при ошибке API.
- `services/first_message.py` не изменён. `handlers/dm/dm_handlers.py` в v1.0.15 изменён только для двух проверок постоянного opt-out: перед постановкой в очередь и прямо перед отправкой.

## v1.0.15 — простой язык и постоянный opt-out

- Максим объясняет проект короткими бытовыми фразами, без расплывчатого слова «подборка» и без обещания «торговать проще».
- Базовая формулировка: есть бесплатная Telegram-группа; программа моментально копирует туда посты из 6 платных закрытых VIP-каналов известных трейдеров; отдельные доступы стоят сотни долларов.

## v1.0.18 — полный аудит и стабилизация

- Исправлена критическая ошибка v1.0.17: обработчик отслеживаемого чата использовал функции аналитики без импортов и мог падать до постановки пользователя в очередь.
- Добавлена защита от параллельного первого DM одному пользователю из нескольких задач одного подключённого аккаунта.
- Проверки opt-out, завершённого и уже активного контакта выполняются повторно непосредственно перед отправкой.
- При завершении диалога старые записи пользователя удаляются из живых очередей этого аккаунта; после админской очистки они не могут отправиться сами без нового сообщения в чате.
- Исправлено состояние после ссылки: первый дополнительный ответ пользователя больше не возвращает цикл из `post_link_active` в обычный `active`.
- Завершённый диалог действительно закрывается после одного финального ответа Максима; поздние обычные сообщения не запускают бесконечные автоответы.
- Статистика ответов учитывает реальный входящий ответ даже при временно выключенном AI-слое.
