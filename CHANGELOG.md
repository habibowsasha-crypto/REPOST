# Changelog

Формат — [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/).

## [Unreleased]

### Added
- UNIQUE-индекс на таблицу групп — дубли теперь отсеиваются на уровне базы.

### Fixed
- SQLite переведён в WAL-режим с `busy_timeout=30000` — больше не ловим `database is locked` при конкурентном доступе из корутин и задач планировщика.
- Утечка Telethon-клиентов: клиенты теперь корректно отключаются при завершении работы бота.
- Очистка `processed_callbacks` вынесена в периодическую задачу планировщика (каждый час) — раньше словарь рос неограниченно.

### Changed
- Асинхронный старт переработан: загрузка сессий и запуск планировщика теперь чётко разделены в `main()`.
- Пример номера телефона в README заменён на международный формат (`+12025550123`).

## [1.0.0]

Первый стабильный релиз.

### Added
- Авторизация нескольких Telegram-аккаунтов прямо в чате бота (телефон → код → 2FA).
- Добавление групп по `@username` или автоимпорт всех групп аккаунта.
- Рассылка сообщений в отдельную группу или сразу по всем с фиксированным или случайным интервалом.
- Поддержка вложений (фото).
- Старт / стоп рассылки по каждой группе независимо.
- История отправок: куда, когда, что и с какого аккаунта.
- Whitelist-доступ через `ADMIN_ID_LIST`.
- SQLite как единственное хранилище — без внешней инфраструктуры.
- Логирование через `loguru` в консоль и файл с ротацией.
- Docker-образ для деплоя.
