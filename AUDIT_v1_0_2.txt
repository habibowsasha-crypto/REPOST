TgBlaster DM AI Dialog v1.0.2 — аудит меню и состояний
Дата проверки: 2026-07-13

Подтверждённая причина ошибок из Telegram:
- старые state-handlers рассылки и групп принимали slash-команды за обычный ввод;
- поэтому /ai_status, /dm_post и другие команды одновременно обрабатывались как команды и как «интервал/группа»;
- главное меню не содержало кнопок DM/AI, хотя обработчики команд были загружены.

Исправления:
- slash-команды исключены из всех соответствующих мастеров ввода;
- добавлен централизованный сброс временных состояний;
- /start и /cancel очищают незавершённые мастера;
- добавлена кнопка сброса текущего ввода;
- добавлены кнопки DM/AI в главное меню;
- добавлена кнопочная остановка активной DM-задачи;
- regex команд закреплены по полной строке;
- .env.example очищен и приведён к Railway-формату без кавычек и секретов.

Проверки:
- AST parse: 34 Python-файла — OK;
- compileall: OK;
- runtime import main/handlers с установленными зависимостями — OK;
- зарегистрировано 68 Telethon handlers — OK;
- обязательные новые menu/DM/AI handlers найдены — OK;
- SQLite DM/AI schema creation и ai_stats — OK;
- централизованный state cleanup — OK;
- group state guards не перехватывают slash-команды — OK;
- OpenAI Responses API path проверен через fake client без сетевого запроса — OK;
- секреты пользователя в архиве не найдены;
- __pycache__, .pyc, .db, .session и лог-файлы удалены.

Ограничения проверки:
- live-вход Telegram, отправка реального DM и реальный запрос OpenAI не запускались;
- для live-теста нужны действующие Telegram-сессии, BOT_TOKEN и оплачиваемый OpenAI API;
- AI_DM_DRY_RUN=true рекомендуется сохранить до проверки логов.
